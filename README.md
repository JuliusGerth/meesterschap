## Week 3 | Carousel omzetten naar React

Deze week ben ik mijn carousel gaan omzetten naar React. Ik heb Embla Carousel gebruikt als basis en mijn bestaande HTML opnieuw opgebouwd als losse componenten. De carousel laat vissoorten zien zoals de Baars, Snoek en Meerval en als je op een vis klikt opent er een dialog met wat statistieken over die soort, zoals hoe vaak die gemeld is, welk aandeel dat is van alle meldingen en welke positie die heeft in de ranglijst. ## Week 4 | Clock component voor de timeline

Deze week ben ik bezig geweest met een klokelement dat laat zien op welke tijden er het meest gemeld wordt. Je ziet een klok en daaronder een tabel met de drukste uren zodat het wat overzichtelijker is dan alleen de klok. ### Feedback

De visualisaties vereisen momenteel te veel scrollen. De voorkeur is dat gebruikers in een oogopslag kunnen zien wat de visualisatie toont en wat de betekenis ervan is, belangrijke informatie moet dus direct zichtbaar zijn zonder veel interactie of navigatie. Daarnaast is de tekst op sommige plekken te klein en mag groter voor betere leesbaarheid. De visualisaties mogen ook eerder in beeld verschijnen en de focus moet duidelijker liggen op de klok en de zonnebloem. Voor vragen kan altijd via e-mail contact worden opgenomen.

## Leerdoelen

### Sprint 0

1. **Eerst een concept maken voor een website**
   Ik wil beter worden in het eerst maken van een concept voordat ik begin.
   Ik begin nu vaak meteen met code en freestyle het dan een beetje, waardoor
   de website met minder kwaliteit wordt opgeleverd.

2. **Animaties maken met keyframes en scroll-animaties**
   Ik wil leren hoe je met keyframes animaties en scroll-animaties maakt.
   Hierdoor worden mijn websites naar een next level gebracht en komen ze niet
   meer zo statisch over.

3. **Mijn ideeën binnen een team laten merken**
   Ik wil leren hoe ik mijn ideeën binnen een team duidelijk kan maken. Ik knik
   vaak ja, ook al heb ik misschien zelf een goed idee.

### Huidige leerdoelen

1. **Beter plannen voor mezelf**
   Ik wil aan het begin van elk project een duidelijke planning maken met
   concrete tussendoelen, zodat ik overzicht houd en niet op het laatste moment
   in tijdnood kom. Dit doe ik door elk project op te splitsen in kleinere taken
   en hier deadlines aan te koppelen.

2. **Effectiever samenwerken**
   Ik wil binnen een team beter communiceren over voortgang en knelpunten, zodat
   we altijd op één lijn zitten en niemand hoeft te wachten of dubbel werk doet.
   Dit doe ik door actief te vragen en te melden wat ik doe, en aan te geven als
   iets niet lukt zodat mijn team me kan helpen.

3. **Documentatie uitgebreider bijhouden**
   Ik houd documentatie nu te beknopt aan — ik voeg zelden plaatjes of code
   snippets toe en schrijf vaak maar een paar zinnen. Ik wil documentatie echt
   als onderdeel van het proces zien, door per project screenshots, relevante
   code en een duidelijke uitleg toe te voegen zodat het ook voor anderen (en
   mijn toekomstige zelf) begrijpelijk is.

## 4. De wiskunde — punten op de klok

```js
function pointOnClock(hour, distance) {
  const angle = (hour / 24) * 2 * Math.PI - Math.PI / 2; // -90° zet uur 0 bovenaan
  return { x: CLOCK_CENTER + distance * Math.cos(angle),
           y: CLOCK_CENTER + distance * Math.sin(angle) };
}
```

`pointOnClock()` ([Clock.jsx:96](src/components/clock/Clock.jsx#L96)) zet een uur + afstand om
naar een x/y-punt met de standaardformule **x = cx + r·cos θ, y = cy + r·sin θ**. De `−π/2`
draait 0 uur naar boven in plaats van naar rechts.

- [MDN — `Math.cos()`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Math/cos) · [`Math.sin()`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Math/sin) · [`Math.PI`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Math/PI)
- [Lippman & Rasmussen — *Points on Circles Using Sine and Cosine*](https://math.libretexts.org/Bookshelves/Precalculus/Book:_Precalculus__An_Investigation_of_Functions_(Lippman_and_Rasmussen)/05:_Trigonometric_Functions_of_Angles/5.03:_Points_on_Circles_Using_Sine_and_Cosine) — de wiskunde achter de formule.

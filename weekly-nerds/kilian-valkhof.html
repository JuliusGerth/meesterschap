Weekly Nerd #3

Kilian Valkhof

Stop Using JavaScript for That — Moving Features from JS to CSS & HTML

The Rule of Least Power

Bedacht door het W3C: als je iets gaat bouwen, moet je kijken of je het kan maken in de minst krachtige taal. Declaratieve talen zoals HTML en CSS zijn hierbij de voorkeur — jij bepaalt het eindproduct en de browser regelt de rest.

Browserfabrikanten luisteren actief naar waar developers JavaScript voor gebruiken, om die gaten te vullen met native HTML en CSS oplossingen.

Het contract met het web: alles wat ooit werkte, moet voor altijd ondersteund blijven worden.

Custom Switches zonder JavaScript

Een custom toggle switch kan je maken met een gewone checkbox:

<label>
    <input type="checkbox" />
    My awesome feature
</label>
input {
    appearance: none; /* verwijdert browser-styling */
}

input:checked {
    background: green;
}

input:checked::before {
    transform: translateX(1rem);
}

Focus zichtbaar maken

input:focus-visible {
    outline: 2px solid dodgerblue;
    outline-offset: 2px;
}

input:focus {
    /* niet: */
    /* outline: none; */

    /* wel: */
    outline-color: transparent;
}

Smooth Scroll zonder JavaScript

html {
    scroll-behavior: smooth;
}

De browser berekent alles zelf, inclusief snelheid en hoe krachtig het apparaat is. Toegankelijker dan JS-scroll, want de browser houdt rekening met systeemvoorkeuren.

@media (prefers-reduced-motion: no-preference) {
    html {
        scroll-behavior: smooth;
    }
}

Zo krijgen alleen gebruikers die géén voorkeur hebben voor minder beweging de animatie te zien.

Scroll margin

#my-target {
    scroll-margin-top: 100px;
}

#my-target:target {
    outline: 10px solid deeppink;
    transition: 1s ease-in-out outline;
}

Accordions met <details>

<details>
    <summary>Hier is mijn titel</summary>
    <p>Lorem ipsum...</p>
</details>
summary::marker {
    font-size: 1.5em;
    content: '▶';
}

[open] summary::marker {
    content: '▼';
}

summary:hover {
    cursor: pointer;
    background: deeppink;
}

Je kunt meerdere <details> elementen dezelfde naam geven zodat er maar één tegelijk open kan zijn — net als een accordion.

Dialogs

<dialog>
    <form method="dialog">
        <h1>Mijn dialog</h1>
    </form>
</dialog>

Het top-layer is een nieuwe laag tussen de website en het scherm van de gebruiker — perfect voor modals en dialogs zonder z-index gedoe.

Field Sizing

input,
textarea {
    field-sizing: content;
}

Hiermee past een input of textarea zich automatisch aan de inhoud aan.

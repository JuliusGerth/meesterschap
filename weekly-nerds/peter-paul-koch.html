Browsers & Support Detection

Wat is een browser?

Een browser bestaat uit de volgende lagen:

  1. HTML
  2. CSS
  3. DOM (Document Object Model)
  4. AOM (Accessibility Object Model)
  5. Interface
  6. JavaScript engine (als aparte component)

In theorie is het mogelijk om dezelfde browser te hebben met een andere JS engine. Een parser is een stukje programma dat een string neemt en analyseert wat er binnen die string staat — dit geldt voor HTML, CSS én JavaScript.

Rendering Engine

De rendering engine is verantwoordelijk voor HTML en CSS parsing en rendering, en voor de constructie van de DOM en AOM trees. De interface en de JS engine vallen hier buiten.

Core vs DOM

  • Core: wat JavaScript een programmeertaal maakt
  • DOM: de manier waarop JS delen van de HTML-pagina kan lezen en aanpassen

Render Blocking

Render blocking betekent dat de browser wacht totdat bepaalde bestanden volledig zijn geladen voordat de pagina wordt gerenderd. CSS en JS (tenzij async of defer) zijn render-blocking.

Defer vs Async

<!-- Laad nu -->
<script src="..."></script>

<!-- Laad later (in volgorde) -->
<script defer src="..."></script>

<!-- Het maakt me niet uit wanneer -->
<script async src="..."></script>

Backward Compatibility

Als iets ooit heeft gewerkt in een browser, moet het op iedere browser ondersteund blijven worden. Dit is het "contract" van het web. Het is dan ook slim om media queries en container queries te gebruiken in plaats van de layout te vaak te veranderen.

Browsergeschiedenis

Een tijdlijn van de belangrijkste browsers:

  • Mosaic — 1993
  • Netscape — 1994 (eerste echte browser)
  • Internet Explorer — 1995
  • Opera — 1995 (beste CSS-ondersteuning, maar klein marktaandeel)
  • IE 5 Mac — 2000
  • Konqueror — 2000 (rendering engine later opgekocht door Apple)
  • Safari — 2003
  • Firefox — 2004 (opvolger van Netscape)
  • Safari iOS — 2007
  • Chrome — 2008
  • Flow — 2020 (eigen browser)
  • Ladybird — 2024
  • Igalia — geen eigen browser, maar groot in open source bijdragen
